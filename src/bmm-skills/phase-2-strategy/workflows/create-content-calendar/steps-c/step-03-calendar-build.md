# Step 3: Calendar Build

**Progress: Step 3 of 3** — Final Step

## STEP GOAL

Generate a week-by-week content calendar with specific content items, channels, and dates.

## MANDATORY RULES

- Calendar must reflect the pillars and channel mapping from Steps 1-2
- Include specific content topics, not just "blog post"
- Balance pillars across the timeframe
- Account for any known dates (holidays, launches, events)

## EXECUTION PROTOCOL

1. Ask: "Are there any key dates to plan around? (product launches, holidays, events, industry conferences)"
2. Generate the calendar using `templates/content-calendar.md`:
   - Week-by-week grid
   - Each entry includes: date, channel, content type, topic/title, pillar, status
   - Balance pillar coverage across weeks
   - Front-load with foundation content if starting from scratch
3. Add production notes:
   - Content that needs to be created from scratch
   - Content that can be repurposed
   - Content that needs subject matter expert input
   - Estimated production effort per piece
4. Present the calendar and ask: "How does this look? Any weeks that need adjustment?"
5. Save to `docs/output/content-calendar.md`

## MENU

- **[C] Complete** — Save and finish
- **[R] Revise** — Adjust calendar entries
- **[B] Back** — Return to Channel Mapping

## SUCCESS METRICS

- Complete calendar for the requested timeframe
- Every entry has channel, topic, and pillar assigned
- Pillar coverage is balanced
- Production effort is realistic
- Saved to `docs/output/`
