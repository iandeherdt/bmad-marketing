# Step 3: Visual Identity

**Progress: Step 3 of 3** — Final Step

## STEP GOAL

Document the visual identity system — colors, typography, imagery style, and usage guidelines.

## MANDATORY RULES

- Be specific — "blue" is not a guideline, "#1a73e8" is
- Include usage rules, not just asset lists
- Visual identity must support the voice and tone from Step 2
- Note: This creates text-based guidelines. For actual asset creation (logos, etc.), the user needs design tools.

## EXECUTION PROTOCOL

1. Ask about existing visual assets:
   - "Do you have an existing logo? Colors? Fonts?"
   - "What's your website URL so I can reference your current visual style?"
2. Define or document **Color Palette**:
   - Primary colors (1-2) with hex codes
   - Secondary colors (2-3) with hex codes
   - Accent colors (1-2) with hex codes
   - Background colors
   - Usage rules: when to use each color
3. Define or document **Typography**:
   - Primary font (headlines)
   - Secondary font (body)
   - Monospace / code font (if applicable)
   - Size hierarchy guidelines
   - Web fonts vs. print fonts
4. Define **Imagery Style**:
   - Photography vs. illustration
   - Style descriptors (lifestyle, editorial, abstract, minimal, etc.)
   - Subject matter guidelines (people, products, abstract)
   - Image do's and don'ts
5. Define **Logo Usage** (if logo exists):
   - Clear space requirements
   - Minimum size
   - Color variations (full color, mono, reversed)
   - Placement guidelines
   - What NOT to do with the logo
6. Define **Layout Principles**:
   - White space philosophy
   - Grid or alignment preferences
   - Content-to-image ratio
7. Save to `docs/output/brand-guidelines.md` using `templates/brand-guidelines.md`
